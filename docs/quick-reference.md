## EN: System Access

### Login Process
1. Navigate to the Finanzas SD application URL
2. Click "Sign In" button
3. Enter your credentials:
   - Email address
   - Password
4. Complete multi-factor authentication (MFA) if prompted
5. You will be redirected to your role's default dashboard

### Supported Browsers
- Google Chrome (recommended)
- Mozilla Firefox
- Microsoft Edge
- Safari (macOS)

### Access Issues
If you cannot log in:
- Verify your credentials are correct
- Check if your account is active (contact your administrator)
- Clear browser cache and cookies
- Try a different browser
- Contact IT support if issues persist

## ES: Acceso al Sistema

### Proceso de Inicio de Sesión
1. Navegue a la URL de la aplicación Finanzas SD
2. Haga clic en el botón "Iniciar Sesión"
3. Ingrese sus credenciales:
   - Dirección de correo electrónico
   - Contraseña
4. Complete la autenticación multifactor (MFA) si se solicita
5. Será redirigido al panel predeterminado de su rol

### Navegadores Compatibles
- Google Chrome (recomendado)
- Mozilla Firefox
- Microsoft Edge
- Safari (macOS)

### Problemas de Acceso
Si no puede iniciar sesión:
- Verifique que sus credenciales sean correctas
- Verifique si su cuenta está activa (contacte a su administrador)
- Limpie la caché y cookies del navegador
- Pruebe con un navegador diferente
- Contacte al soporte de TI si los problemas persisten

## EN: User Roles

The system supports four main user roles:

| Role | Description | Access Level |
|------|-------------|--------------|
| **SDMT** | Service Delivery Management Team | Access to cost catalog, forecasting, and budget management |
| **VENDOR** | External Vendors | Limited access to catalog and assigned projects |
| **EXEC_RO** | Executive Read-Only | View-only access to dashboards and reports |

### Switching Roles
If you have multiple roles:
1. Click on your role badge in the top navigation bar
2. Select desired role from dropdown menu
3. System will redirect to appropriate dashboard

## ES: Roles de Usuario

El sistema admite cuatro roles principales de usuario:

| Rol | Descripción | Nivel de Acceso |
|-----|-------------|-----------------|
| **SDMT** | Equipo de Gestión de Entrega de Servicios | Acceso a catálogo de costos, pronósticos y gestión de presupuesto |
| **VENDOR** | Proveedores Externos | Acceso limitado al catálogo y proyectos asignados |
| **EXEC_RO** | Ejecutivo Solo Lectura | Acceso de solo lectura a paneles e informes |

### Cambio de Roles
Si tiene múltiples roles:
1. Haga clic en su insignia de rol en la barra de navegación superior
2. Seleccione el rol deseado del menú desplegable
3. El sistema lo redirigirá al panel apropiado

## EN: Troubleshooting

### Issue: Page Not Loading
**Symptoms**: Blank screen or loading spinner that never completes

**Solutions**:
1. Refresh the page (F5 or Ctrl+R)
2. Check your internet connection
3. Clear browser cache
4. Disable browser extensions
5. Try incognito/private mode

### Issue: Data Not Saving
**Symptoms**: Save button clicked but data not persisted

**Solutions**:
1. Check for error messages
2. Verify all required fields are filled
3. Check for validation errors (red highlights)
4. Wait for any pending operations to complete
5. Try logging out and back in

### Issue: Permission Denied
**Symptoms**: "You don't have permission" message appears

**Solutions**:
1. Verify you're using the correct role
2. Switch to appropriate role if you have multiple
3. Contact your administrator if role is incorrect
4. Check if your account has been deactivated

## ES: Solución de Problemas

### Problema: Página No Carga
**Síntomas**: Pantalla en blanco o spinner de carga que nunca termina

**Soluciones**:
1. Actualice la página (F5 o Ctrl+R)
2. Verifique su conexión a internet
3. Limpie la caché del navegador
4. Deshabilite extensiones del navegador
5. Pruebe modo incógnito/privado

### Problema: Datos No se Guardan
**Síntomas**: Botón de guardar clickeado pero datos no se mantienen

**Soluciones**:
1. Busque mensajes de error
2. Verifique que todos los campos requeridos estén completos
3. Busque errores de validación (resaltados en rojo)
4. Espere a que las operaciones pendientes se completen
5. Intente cerrar sesión y volver a iniciar

### Problema: Permiso Denegado
**Síntomas**: Aparece mensaje "No tiene permiso"

**Soluciones**:
1. Verifique que esté usando el rol correcto
2. Cambie al rol apropiado si tiene múltiples
3. Contacte a su administrador si el rol es incorrecto
4. Verifique si su cuenta ha sido desactivada
